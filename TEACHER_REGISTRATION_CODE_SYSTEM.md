# Teacher Registration Code System

## Overview

Teachers now register using a unique registration code that automatically assigns their subjects and form class allocations. The code is generated by admins, distributed to teachers via email, and used during registration to pre-populate all assignments.

## Key Features

✅ **Unique Registration Codes**: Admin-generated codes for each teacher  
✅ **Auto-Assigned Allocations**: Subjects and form class automatically applied  
✅ **Code Expiration**: Optional expiry dates to manage code distribution  
✅ **Code Revocation**: Admins can revoke unused codes  
✅ **Email Distribution**: Built-in email sending to teachers  
✅ **One-Time Use**: Codes marked as used after registration  
✅ **Activity Logging**: All code operations tracked for audit  

## System Components

### 1. Registration Code Utilities (`registrationCodeUtils.js`)

- `generateSimpleRegistrationCode(database)`: Creates sequential codes in format `tch-2026-NNN` (e.g., tch-2026-001, tch-2026-002)
- `isValidCodeFormat(code)`: Validates code format matches `tch-2026-NNN` pattern
- Uses database counter to ensure unique sequential numbering
- Counter stored in `ids.registrationCodes` document

### 2. Registration Codes Repository (`repos/registrationCodes.js`)

Database operations for registration codes:
- `createRegistrationCode()`: Store code with allocations
- `getRegistrationCodeByCode()`: Retrieve code details
- `isCodeValid()`: Check if code can be used (not used, not revoked, not expired)
- `markCodeAsUsed()`: Record code usage
- `revokeRegistrationCode()`: Invalidate code
- `listRegistrationCodes()`: List all codes with filters

### 3. Admin Endpoints

#### Generate Registration Code
**POST** `/api/admin/registration-codes`

```json
{
  "displayName": "John Smith",
  "email": "john@school.com",
  "subjectIds": ["math-001", "english-002"],
  "formClassId": "jss-1a",
  "expiryDays": 30
}
```

**Response:**
```json
{
  "success": true,
  "message": "✅ Registration code generated successfully",
  "code": {
    "code": "tch-2026-001",
    "displayName": "John Smith",
    "email": "john@school.com",
    "subjectsCount": 2,
    "formClassAssigned": true,
    "expiresAt": "2026-07-04T...",
    "createdAt": "2026-06-04T...",
    "status": "ACTIVE"
  }
}
```

#### List Registration Codes
**GET** `/api/admin/registration-codes`

Query parameters:
- `status`: ACTIVE, USED, EXPIRED, REVOKED
- `used`: true/false

**Response:**
```json
{
  "total": 5,
  "codes": [
    {
      "code": "ABCD-EFGH-IJKL",
      "displayName": "John Smith",
      "email": "john@school.com",
      "subjectsCount": 2,
      "formClassAssigned": true,
      "status": "ACTIVE",
      "used": false,
      "usedBy": null,
      "usedAt": null,
      "createdAt": "2026-06-04T...",
      "expiresAt": "2026-07-04T..."
    }
  ]

```

#### Get Code Details
**GET** `/api/admin/registration-codes/:code`

Returns full details of a registration code including subject IDs.

#### Revoke Code
**DELETE** `/api/admin/registration-codes/:code`

Only works for unused codes. Marks code status as REVOKED.

#### Send Code to Email
**POST** `/api/admin/registration-codes/:code/send`

```json
{
  "recipientEmail": "john@school.com"
}
```

Sends professional email with code and registration instructions.

### 4. Teacher Registration Endpoint

**POST** `/api/auth/register/teacher`

```json
{
  "email": "john@school.com",
  "password": "SecurePass123",
  "displayName": "John Smith",
  "registrationCode": "ABCD-EFGH-IJKL"
}
```

**Flow:**
1. Validates registration code
2. Checks code hasn't been used or revoked
3. Verifies email matches code (if pre-assigned)
4. Creates user account with allocations from code
5. Marks code as used
6. Sends welcome email with allocation summary

**Response:**
```json
{
  "success": true,
  "message": "✅ Account created successfully! You can now login with your credentials.",
  "user": {
    "username": "tch-2026-001",
    "email": "john@school.com",
    "displayName": "John Smith",
    "role": "TEACHER",
    "portal": "TEACHER",
    "subjectsAssigned": 2,
    "formClassAssigned": true
  }
}
```

## Admin Workflow

### Step 1: Generate Code
```bash
POST /api/admin/registration-codes
{
  "displayName": "Sarah Johnson",
  "email": "sarah@school.com",
  "subjectIds": ["mathematics", "physics"],
  "formClassId": "ss1-b",
  "expiryDays": 14
}
```

**Response:** Code generated (e.g., `WXYZ-MNOP-QRST`)

### Step 2: Send Code to Teacher
```bash
POST /api/admin/registration-codes/WXYZ-MNOP-QRST/send
{
  "recipientEmail": "sarah@school.com"
}
```

Teacher receives email with:
- Registration code
- List of assigned subjects (count)
- Form class assignment info
- Registration instructions
- Link to registration page

### Step 3: Teacher Registers
Teacher visits registration page and enters:
- Full name: `Sarah Johnson`
- Email: `sarah@school.com`
- Password: `SomePassword123`
- Registration Code: `WXYZ-MNOP-QRST`

System automatically:
- Creates account with username `tch-2026-002`
- Assigns 2 subjects (Mathematics, Physics)
- Assigns form class (SS1-B)
- Marks code as used
- Sends welcome email

## Code Lifecycle

### ACTIVE
- Initial state when created
- Can be used for registration
- Can be revoked
- Can be re-sent

### USED
- Code has been used by a teacher
- Cannot be revoked (teacher is registered)
- Cannot be used again
- Tracks which teacher used it and when

### REVOKED
- Manually revoked by admin
- Cannot be used for registration
- Only possible if code hasn't been used
- Used when code is no longer needed (e.g., teacher enrolled differently)

### EXPIRED
- Automatically expired after expiry date
- Cannot be used for registration
- Admin can generate new code if needed

## Validation Rules

### Code Format
- Format: `tch-2026-NNN` where NNN is a 3-digit number (001-999)
- Example codes: tch-2026-001, tch-2026-042, tch-2026-100
- Case-insensitive (stored uppercase: TCH-2026-001)
- Sequential numbering from admin generation

### Code Validation
Must pass ALL checks to be valid:
1. Code exists in database
2. Status is ACTIVE
3. Not already used
4. Not revoked
5. Not expired (if expiry date set)

### Registration Validation
1. Valid registration code required
2. Email required and valid format
3. Password minimum 6 characters
4. Email must match code (if pre-assigned to code)
5. Email not already registered

## Example Workflows

### Workflow 1: Bulk Teacher Onboarding

1. Admin has list of new teachers with their subject allocations
2. Admin creates registration code for each teacher:
   ```
   POST /api/admin/registration-codes
   {
     "displayName": "Teacher Name",
     "email": "teacher@school.com",
     "subjectIds": [...],
     "formClassId": "..."
   }
   ```
3. Admin sends code to teacher:
   ```
   POST /api/admin/registration-codes/{code}/send
   ```
4. Teacher receives email with code and registration link
5. Teacher completes registration with code
6. System auto-assigns all subjects and form class
7. Teacher can immediately start entering results

### Workflow 2: Code Expiry Management

1. Admin generates codes with 30-day expiry
2. Teachers receive notifications with deadline
3. Expired codes automatically become invalid
4. If teacher doesn't register in time:
   - Admin can revoke old code and generate new one
   - Re-send to teacher with new deadline

### Workflow 3: Code Distribution via CSV

1. Admin creates spreadsheet with teachers and allocations
2. System generates codes in bulk
3. Codes exported to CSV with teacher emails
4. Admin sends emails manually (or auto-trigger)
5. Teachers register using codes

## Troubleshooting

**Issue**: Code says invalid when it should be valid
- Check code hasn't been used
- Check code hasn't been revoked
- Check code hasn't expired
- Verify code format is correct

**Issue**: Teacher gets email mismatch error
- Code was generated with specific email
- Teacher trying to register with different email
- Solution: Generate new code or let teacher use pre-assigned email

**Issue**: Code appears used but teacher never completed registration
- If teacher registration partially failed, code might be marked used
- Contact admin to revoke and regenerate
- Consider implementing retry mechanism
